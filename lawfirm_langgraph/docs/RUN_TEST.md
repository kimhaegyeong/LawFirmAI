# 테스트 실행 가이드

## 빠른 테스트 실행

PowerShell에서 다음 명령어를 실행하세요:

```powershell
cd D:\project\LawFirmAI\LawFirmAI\lawfirm_langgraph
.venv\Scripts\python.exe test_manual.py
```

## 테스트 스크립트 목록

1. **test_manual.py** - 단계별 수동 테스트 (권장)
   - 단계별로 테스트를 실행하여 어디서 문제가 발생하는지 확인 가능
   
2. **test_workflow.py** - 종합 워크플로우 테스트
   - 모든 테스트를 한 번에 실행
   
3. **test_quick.py** - 빠른 기본 기능 테스트
   - Import, 설정, Graph/App 생성만 확인

## 문제 해결

### Import 오류

```
✗ Import 테스트 오류: No module named 'xxx'
```

**해결 방법:**
```powershell
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 설정 오류

```
✗ 설정 로딩 테스트 오류: ...
```

**해결 방법:**
- 상위 프로젝트의 `.env` 파일 확인
- 필요한 환경 변수 설정 확인

### 워크플로우 실행 오류

```
⚠ 워크플로우 실행 실패: ...
```

**참고:**
- 이것은 LLM API 키 설정 문제일 수 있습니다
- Graph와 App 생성이 성공했다면 정상입니다
- LangGraph Studio를 사용할 수 있습니다

## 다음 단계

모든 기본 테스트(워크플로우 실행 제외)가 통과하면:

```powershell
langgraph dev
```

브라우저에서 `http://localhost:8123`에 접속하여 LangGraph Studio를 사용할 수 있습니다.
